#+AUTHOR: Michael Jones (YardQuit)

* Overview

Donkey is an opinionated modal editing minor-mode for Emacs, that
leverages Emacs' own built-in functions wherever possible. There is no
overlay state — just a minor-mode keymap that toggles on and off.

** Design Philosophy

- Leverage Emacs native commands and built-in functions wherever possible
- Custom commands only where beneficial
- No overlay-based state tracking — just minor-mode keymaps
- Graceful handling of optional dependencies (smartparens, org-mode)
- Cross-platform support (Linux/Wayland/X11, macOS, Windows)
- Terminal mode (=emacs -nw=) support with DECSCUSR cursor sequences

* Installation

Donkey isn't on MELPA, so pick whichever of these fits your setup.
All are equivalent; they just differ in how =donkey.el= gets onto
disk and onto your =load-path=.

** Manual

Place =donkey.el= at =donkey/donkey.el= inside your
=user-emacs-directory= (run =C-h v user-emacs-directory= to see
where that is on your system — typically =~/.config/emacs/= or
=~/.emacs.d/= on Linux and macOS, or the equivalent under =%APPDATA%=
on Windows), then add the following to your =init.el= or =config.el=:

#+begin_src emacs-lisp

  (let ((donkey-path (expand-file-name "donkey/donkey.el"
                                     user-emacs-directory)))
    (if (file-exists-p donkey-path)
        (load donkey-path nil t)
      (message "WARNING: Donkey modal module not found at %s." donkey-path)))

  (donkey-mode 1)

#+end_src

** use-package with a manual clone

Clone the repository into =donkey/= inside your
=user-emacs-directory=, then point =use-package= at it with
=:load-path=. A relative path is resolved against
=user-emacs-directory= automatically:

#+begin_src emacs-lisp

  (use-package donkey
    :load-path "donkey"
    :config
    (donkey-mode 1))

#+end_src

** use-package with =straight.el=

#+begin_src emacs-lisp

  (use-package donkey
    :straight (:host github :repo "yardquit/donkey")
    :config
    (donkey-mode 1))

#+end_src

** use-package with =quelpa=

The =:quelpa= keyword isn't built into =use-package= itself — it's
added by the separate =quelpa-use-package= package, so both =quelpa=
and =quelpa-use-package= need to be installed first:

#+begin_src emacs-lisp

  (use-package donkey
    :quelpa (donkey :fetcher github :repo "yardquit/donkey")
    :config
    (donkey-mode 1))

#+end_src

* Usage

- Press =C-g= to enter DONKEY Normal state.
- In Normal: =h=, =j=, =k=, =l= navigate; =i=, =I=, =a=, =A=, =o=, =O=, =c= enter Insert state.
- In Insert: Standard Emacs behavior, press =C-g= to return to Normal.
- State indicators show in modeline: =DONKEY[N]= = Normal, =DONKEY[I]= = Insert.

* Customizing Keybindings

You can override existing bindings or add new ones in your =config.el=
by modifying =donkey-normal-mode-map=.

** Override an existing binding:

#+begin_src emacs-lisp

  ;; Replace the default behavior of ';' with consult-outline
  (when (boundp 'donkey-normal-mode-map)
    (keymap-set donkey-normal-mode-map ";" #'consult-outline))

#+end_src

** Unbind a suppressed key to fall through:

#+begin_src emacs-lisp

  ;; Make ',' fall through to the major mode instead of being ignored
  (when (boundp 'donkey-normal-mode-map)
    (keymap-unset donkey-normal-mode-map ","))

#+end_src

** Add a leader key map:

#+begin_src emacs-lisp

  ;; Leader Map Definition
  (when (boundp 'donkey-normal-mode-map)
    (unless (boundp 'donkey-leader-map)
      (defvar donkey-leader-map (make-sparse-keymap)
        "Leader key map under SPC."))
    (keymap-set donkey-normal-mode-map "SPC" 'donkey-leader-map))

#+end_src

* Excluded Modes

DONKEY Normal state is permanently disabled in certain major modes where
key suppression would break core functionality. In these modes, DONKEY
stays in Insert state (passthrough), keeping DONKEY active but
non-interfering.

** Built-in Defaults

The following modes are excluded by default because they manage
subprocess interaction or terminal emulation where suppressing keys
via =suppress-keymap= would break functionality:

| Mode          | Reason                                                                 |
|---------------+------------------------------------------------------------------------|
| =comint-mode= | Process interaction — all input goes to subprocess. Covers derivatives |
|               | like =shell-mode=, =inferior-python-mode=, =ielm-mode=, =gud-mode=     |
|               | via =derived-mode-p=.                                                  |
| =term-mode=   | Full PTY terminal emulation — character-at-a-time, raw mode.           |
| =vterm-mode=  | Same as =term-mode= but faster implementation.                         |
| =eshell-mode= | Own command-line parser and input loop.                                |

Derived modes are caught automatically by the =derived-mode-p= check
in =donkey--excluded-mode-p= (used consistently everywhere DONKEY needs
to know whether a buffer is excluded), so listing =comint-mode= alone
covers dozens of comint derivatives.

** Adding Additional Excluded Modes

Modes like =dired-mode=, =ibuffer-mode=, =magit-status-mode=, and
=epa-key-list-mode= are NOT excluded by default. In these modes, DONKEY
Normal state works (keys are suppressed, =hjkl= navigates), but some
users may prefer passthrough to retain native keybindings.

To add modes to the exclusion list in your =config.el=:

#+begin_src emacs-lisp

  (with-eval-after-load 'donkey
    (dolist (mode '(dired-mode ibuffer-mode magit-status-mode epa-key-list-mode))
      (add-to-list 'donkey-excluded-modes mode)))

#+end_src

Or add a single mode:

#+begin_src emacs-lisp

  (with-eval-after-load 'donkey
    (add-to-list 'donkey-excluded-modes 'pdf-view-mode))

#+end_src

Verify the current exclusion list:

#+begin_src emacs-lisp

  M-x eval-expression RET donkey-excluded-modes RET

#+end_src

* Enter Key Rules (DWIM)

DONKEY's =donkey-enter-dwim= command dispatches the Enter (RET) key based
on context. In editing modes, Enter is blocked to prevent accidental
line breaks. In non-editing modes, the native RET binding is invoked.

For =org-mode= and =markdown-mode=, DONKEY uses a rule-based dispatch
system to execute context-aware actions when Enter is pressed.

** Built-in Default Rules

The following rules are installed by default (controlled by
=donkey-default-enter-rules-enabled=, which defaults to =t=):

#+begin_src emacs-lisp

  ;; Toggle checkboxes in org list items
  (donkey-add-enter-rule item :checkbox org-toggle-checkbox)

  ;; Toggle TODO/DONE on org headlines
  (donkey-add-enter-rule headline :todo-type donkey-org-todo)

  ;; Follow links in org, markdown, or plain text
  (donkey-add-enter-rule link nil org-open-at-point
                       markdown-follow-thing-at-point
                       browse-url-at-point)

#+end_src

Each rule specifies:
- =ELEMENT-TYPE= :: Org element type to match (e.g. =item=, =headline=, =link=)
- =PROPERTY= :: Org element property to check (e.g. =:checkbox=, =:todo-type=), or =nil= for no property check
- =COMMANDS= :: One or more commands tried in sequence (fallback chain). The first =fboundp= and =commandp= match wins.

** Rule Priority

Rules use =add-to-list= which prepends to the front of the list. The
first matching rule wins.

To prioritize a specific command for an element type, add it LAST in
your =config.el= so it appears at the front of the dispatch list:

#+begin_src emacs-lisp

  ;; These will be checked in this order:
  ;; 1. shr-follow-link-at-point (added last, checked first)
  ;; 2. org-open-at-point (added first, checked second)
  (donkey-add-enter-rule link nil org-open-at-point)
  (donkey-add-enter-rule link nil shr-follow-link-at-point)

#+end_src

To replace a default rule entirely, disable the built-ins first, then
add your own:

#+begin_src emacs-lisp

  (with-eval-after-load 'donkey
    (setq donkey-default-enter-rules-enabled nil)
    (donkey-add-enter-rule link nil shr-follow-link-at-point))

#+end_src

** Disabling Built-in Rules

To define your own rules entirely, disable the defaults:

#+begin_src emacs-lisp

  (with-eval-after-load 'donkey
    (setq donkey-default-enter-rules-enabled nil))

#+end_src

** Adding Custom Enter Rules

Rules can be added from =config.el= using the =donkey-add-enter-rule=
macro. Example:

#+begin_src emacs-lisp

  (with-eval-after-load 'donkey
    ;; Follow links with shr-follow-link-at-point as first choice
    (donkey-add-enter-rule link nil shr-follow-link-at-point
                         org-open-at-point
                         browse-url-at-point)

    ;; Open agenda files from org-agenda table view
    (donkey-add-enter-rule org-agenda-heading nil
                         org-agenda-do-default-action))

#+end_src

** Dispatch Order

=donkey-enter-dwim= checks elements in this order:

1. =org-agenda-mode= RET binding (if in agenda)
2. Context element (most specific, e.g. a link inside a headline)
3. Parent element (e.g. the headline itself)
4. Ancestor elements (walking up the org element tree)
5. Non-editing mode native RET binding

All rules are checked at each level before moving to the next. This
ensures the most specific match fires first.

** Available Element Types

Common org element types for rules:

| Element Type | Useful Properties | Example Commands                           |
|--------------+-------------------+--------------------------------------------|
| =link=       | =nil=             | =org-open-at-point=, =browse-url-at-point= |
| =headline=   | =:todo-type=      | =org-todo=, =donkey-org-todo=                |
| =item=       | =:checkbox=       | =org-toggle-checkbox=                      |
| =src-block=  | =nil=             | =org-edit-special=                         |
| =table-row=  | =nil=             | =org-table-next-row=                       |
| =plain-list= | =nil=             | =org-toggle-item=                          |

* Cursor Management

DONKEY manages cursor shapes independently for Normal and Insert states.
In terminal mode (=emacs -nw=), DECSCUSR escape sequences are used for
reliable cursor rendering across terminal emulators.

** Configuration

#+begin_src emacs-lisp

  (with-eval-after-load 'donkey
    (setq donkey-cursor-normal 'box      ; box, bar, hollow, or nil for global default
          donkey-cursor-insert '(bar . 2))) ; bar with width 2

#+end_src

** Terminal Compatibility

Some terminals do not support DECSCUSR or produce garbled output. DONKEY
maintains a denylist of terminal types where escape sequences are
suppressed:

#+begin_src emacs-lisp

  ;; View current denylist
  M-x eval-expression RET donkey-decscusr-denied-terminals RET

  ;; Add a terminal to the denylist interactively
  M-x donkey-add-denylist-entry RET

  ;; Remove a terminal from the denylist
  M-x donkey-remove-denylist-entry RET

#+end_src

Default denied terminals: =dumb=, =linux= (framebuffer console).

** Platform Diagnostics

Run =M-x donkey-debug-platform= for a detailed report of:
- System type and Emacs version
- Display backend (GUI vs terminal)
- TTY type and TERM environment variable
- Clipboard tool availability
- Native compilation status

* Clipboard Integration

DONKEY's =donkey-yank= and =donkey-yank-pop= commands integrate with the
system clipboard, falling back to the kill ring when no clipboard tools
are available.

** Detection

DONKEY checks for clipboard tool availability fresh each time it's
needed (not cached), since the answer can differ per frame — a single
=emacs --daemon= process can have both a GUI frame (=emacsclient -c=)
and a terminal frame (=emacsclient -t=) at once, each with different
capabilities. Detection covers:
- =wl-copy= / =wl-paste= (Wayland)
- =xclip= / =xsel= (X11)
- =pbcopy= / =pbpaste= (macOS)
- Native GUI clipboard (graphical Emacs)
- Windows native clipboard

If no tools are found and Emacs is running in terminal mode, a warning
is shown once at load time. Yank operations fall back to the kill ring
silently after the first warning.

* Input Method Management

DONKEY automatically saves and restores the active input method when
transitioning between Normal and Insert states. This is useful for
non-English keyboard layouts (e.g. Swedish Postfix).

- Entering Normal state: saves and deactivates the current input method
- Entering Insert state: restores the previously saved input method

This prevents input method key sequences from interfering with Normal
mode keybindings.

* Smartparens Integration

DONKEY always, with no configuration needed:
- Clears transient smartparens overlays on Insert-to-Normal transition
- Defers overlay cleanup via idle timer to avoid race conditions

Call =(donkey-setup-smartparens)= in your config after loading
=smartparens= to additionally bind =C-g= in =smartparens-mode-map= and
its overlay keymaps. This binding is opt-in and not installed
automatically. If smartparens is not installed, DONKEY functions
normally without errors.

These bindings live in Smartparens' own keymaps, independent of
=donkey-mode='s lifecycle, so they are not undone when =donkey-mode=
is disabled. That's safe: with =donkey-mode= off, pressing =C-g=
through one of these bindings falls through to =keyboard-quit=
instead of re-enabling DONKEY's Normal state.

#+begin_src emacs-lisp

  (with-eval-after-load 'smartparens
    (donkey-setup-smartparens))

#+end_src

Or with =use-package=, in =smartparens='s =:config= block, which runs
at the same point as =with-eval-after-load=:

#+begin_src emacs-lisp

  (use-package smartparens
    :ensure t
    :config
    (donkey-setup-smartparens))

#+end_src

** Wrapping a Selection with a Delimiter

Select a region (=v=/=V=), then press one of =(=, =[=, ={=, ="=, ='=,
=`= while still in Normal state to wrap it — e.g. selecting =word=
and pressing =(= yields =(word)= when Smartparens is loaded. This
works without leaving Normal state stuck in Insert afterward.

Under the hood, =donkey-normal-mode-map= normally suppresses every
printable key so it can't accidentally self-insert. These delimiters
are the one exception, and only while a region is active:
`donkey-wrap-region` momentarily enters Insert state *without*
clearing the selection first (unlike =i=/=a= and the other Insert
Mode Entry commands), runs the ordinary character insertion so
Smartparens' own region-wrap logic can see the still-active region,
then returns to Normal state automatically. With no region active,
the key does nothing, same as any other suppressed key in Normal
state.

The delimiter set is =donkey-wrap-delimiters= (a Customization Group
variable); change it before =donkey.el= loads to add or remove
characters.

* Position Tracking

=M-x donkey-jump-back= (bound to =S=) rotates through a ring of
previously recorded cursor positions in the current buffer. Positions
are tracked automatically via =post-command-hook= whenever the cursor
moves.

- Ring size controlled by =donkey-position-ring-max= (default: 10)
- Ring is buffer-local — each buffer keeps its own history
- Independent of the mark ring and region

To keep more (or fewer) positions, set =donkey-position-ring-max= in
your =init.el= or =config.el=, e.g. after the =load= call from
Installation:

#+begin_src emacs-lisp

  (setq donkey-position-ring-max 20)

#+end_src

Unlike =donkey-wrap-delimiters=, this takes effect immediately —
`donkey--track-position` reads it fresh on every call, so it can be
changed at any time, not just before =donkey.el= loads.

* Org-Scratch Buffer

=M-x donkey-org-scratch= creates or switches to a scratch buffer in
=org-mode=. Useful for quick note-taking without creating a file.
